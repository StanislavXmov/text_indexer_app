## test_get_answer

Небольшой тестовый проект на FastAPI.

Что делает:
- поднимает HTTP API;
- отдает простую HTML-страницу на `GET /` (проверка, что приложение работает);
- предоставляет endpoint `GET /health`, который возвращает `{"status": "ok"}` для health-check.

Запуск в режиме разработки:

`uv run uvicorn get_answer.api:app --reload`

## Что делает `get_answer/main.py`

`get_answer/main.py` — это консольная точка входа для тестового запуска пайплайна.

Файл:
- создает конфиг `RAGConfig`;
- берет тестовый текст `SAMPLE_TEXT`;
- вызывает `create_pdf(...)` и сохраняет результат в путь из конфига (`config.output_pdf_path`);
- измеряет и выводит время выполнения.

Внутри также есть закомментированный пример запуска RAG-запроса через `RAGPipeline` (его можно включить при необходимости).
